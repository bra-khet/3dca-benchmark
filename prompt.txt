Create a single self-contained HTML file (vanilla JS + WebGL or Canvas/WebGL with a fragment shader) that implements a creative 3D raymarched metaball cellular growth system inspired by cellular automata.

Key requirements:
- Simulation (in JS): Maintain an array of "blobs" in 3D space. Each has position (vec3), radius, and energy (float). Each frame, for every blob, count neighbors within a tunable radius. Apply simple local rules: 
  - If neighbor count and energy are high enough → gain energy or spawn a small new blob nearby with some randomness.
  - If isolated or low energy → lose energy and shrink (remove if below threshold).
  - Nearby blobs can influence/merge energy slightly.
  Use a spatial hash or simple O(n^2) for small-to-medium count (start with 40-80 blobs, make it tunable).
- Rendering: Full raymarching in a fragment shader. Scene is the smooth union (smoothMin / polynomial smooth minimum) of metaball spheres. Add:
  - Nice lighting (key directional light + fill/ambient + subtle rim or emissive based on energy).
  - Cheap ambient occlusion or shadow approximation if feasible.
  - Emissive glow scaled by energy (brighter/more saturated when active).
  - Subtle surface detail via cheap noise or time-based displacement.
  - Color mapping: energy and/or "age" drive a nice organic palette (e.g. deep blues/purples base with cyan/magenta/yellow glows — make palette easy to tweak).
- Animation & evolution: Continuous or stepped time. Interpolate blob states smoothly for fluid motion. Slow auto-orbit camera + full mouse orbit/look controls. Add simple trails or fading history for visual richness.
- Interactivity & polish: Sliders or dat.GUI-style controls for rule parameters (neighbor radius/threshold, growth rate, death rate, spawn chance, max blobs). Pause/step/speed controls. Reset button. Display current generation or "alive" count. Target smooth 60fps. Handle window resize. Add subtle post effects if easy (vignette, basic glow).
- Code quality: Clean structure (simulation logic separate from rendering). Well-commented key sections (especially SDF math and rule logic). Immediately runnable by saving as .html. No external dependencies except possibly a small CDN for dat.GUI if it improves UX (or pure vanilla sliders).
- Performance & stability: Reasonable raymarch step count. Stable long-term behavior from the rules (not immediate extinction or total fill — tune defaults accordingly). Avoid artifacts.

Make it visually striking and "alive" — organic merging, pulsing clusters, emergent structures. Prioritize high visual payoff while keeping the code reliable and not overly long. Add a short comment at the top with tunable parameters and how the rules work.
